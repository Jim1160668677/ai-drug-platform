# AI模式精准药物设计系统 — 实施计划

## Context

用户基于 GitLab 联合创始人 Sid 的个性化癌症治疗经历，希望构建一个"AI模式精准药物设计系统"，实现"一人一药"的个性化精准治疗。项目目录当前为空（仅有需求文档），需从零搭建完整系统。

**用户确认的关键决策**：
- 实现范围：完整系统全量实现（3个阶段全部覆盖）
- 前端：Next.js + React + TailwindCSS
- 基础设施：Docker Compose 一键编排
- 外部API：真实调用 + Mock 双模式（默认 Mock，配置 key 后真实调用）

**实施策略**：由于工作量巨大，分阶段实施。本次先完成「第一阶段 P0 全功能 + 项目骨架（含二、三阶段代码框架）」，确保系统可运行、可演示。二、三阶段的深度功能（DeepChem/PyG/DiffDock/Flower/PySyft）提供集成代码框架，部分依赖 GPU/外部环境由用户后续配置。

### 第一阶段 P0 — 搭建骨架 + 核心功能（可运行可演示）

## 核心设计要点

### Mock/Real 双模式
- `config.py` 的 `USE_MOCK: bool = True` 默认 Mock
- `core/deps.py` 的 `get_llm_client()/get_gene_client()/...` 工厂依据开关注入
- 所有 service 层只依赖抽象接口，对上层透明切换
- `.env` 配 `USE_MOCK=false` + `OPENAI_API_KEY` 即启用真实调用

### API 路由（RESTful）
```
POST /api/v1/auth/login
GET/POST /api/v1/projects
POST /api/v1/data/upload          # 分片上传,返回dataset_id
GET  /api/v1/data/{id}            # 含质量报告
POST /api/v1/data/{id}/parse      # 触发解析(异步job)
POST /api/v1/targets/discover     # 输入dataset_id,返回靶点列表(含证据分级)
GET  /api/v1/targets/{id}
POST /api/v1/targets/{id}/repurpose
POST /api/v1/molecules/design     # P2
POST /api/v1/molecules/{id}/dock  # P2
POST /api/v1/knowledge/gene
POST /api/v1/knowledge/variant
POST /api/v1/chat                 # 自然语言问答(分级路由)
GET/POST /api/v1/hypotheses       # 多假设并行
POST /api/v1/hypotheses/{id}/analyze
GET/POST /api/v1/experiments
POST /api/v1/reports/{project_id}/sdtm
POST /api/v1/workflows/run
GET  /api/v1/audit/logs
```

### Docker Compose 服务（11个）
backend / worker（异步任务） / frontend / postgres（+TimescaleDB） / redis / chromadb / neo4j / minio / nextflow / nginx / flower-server（第三阶段）

### 关键数据库表
users（5角色RBAC）/ projects / datasets（quality_json） / targets（evidence_grade I-IV） / molecules / treatments / hypotheses / experiments / analysis_jobs（cost_usd/duration/层级） / audit_logs（append-only） / workflow_runs

## 验证方法（第一阶段 MVP）

1. `make up` 启动全部服务，`docker compose ps` 全部 healthy
2. `make migrate && make seed` 建表 + 灌入样本患者/数据
3. 访问 `http://localhost` 前端登录（创始人角色）
4. 上传样本 RNA-seq CSV → 触发 `parse` → 质量报告返回
5. 调 `POST /targets/discover` → 返回含 ClinVar/COSMIC 注释 + 证据分级的靶点列表
6. 调 `POST /targets/{id}/repurpose` → 返回 ChEMBL 老药新用候选
7. 调 `POST /chat` 提问"EGFR T790M 有哪些靶向药?" → 分级路由返回
8. `POST /reports/{id}/sdtm` → 下载 CDISC SDTM 文件
9. 创建 2 个 Hypothesis 并行分析，对比看板可见差异
10. `.env` 设 `USE_MOCK=false` + `OPENAI_API_KEY` → 重复步骤 5-7 走真实链路
11. `make test` 后端关键链路测试通过
