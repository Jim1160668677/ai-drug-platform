### 功能 1：AI 医学红线规则（P0）

#### 需求来源
v3.0 文档第 11.3 节定义了 5 条医学特定红线规则，当前 `guardrail.py` 仅有 4 类基础护栏（越狱/剂量/PII/绝对化），缺少医学领域特定红线。

#### 5 条医学红线规则
1. **诊断请求拦截**：用户要求"诊断我是否患癌"等 → 拦截 + 提示就医
2. **用药建议拦截**：用户要求"推荐我吃什么药"等 → 拦截 + 提示咨询医生
3. **预后预测标注**：输出含"预计生存期 X 个月"等 → 添加免责声明
4. **矛盾结论警告**：输出与已知医学共识矛盾 → 添加警告标注
5. **证据不足标注**：输出基于低证据等级（Grade C/D）的结论 → 添加"证据有限"标注

#### 文件变更

**文件 1：`frontend/lib/hooks/useTaskProgress.ts`（新建）**

- **What**：React Hook 封装 WebSocket 连接 + 自动重连 + HTTP 回退
- **Why**：提供统一的任务进度订阅接口
- **How**：
  ```typescript
  export function useTaskProgress(taskId: string | null) {
    const [progress, setProgress] = useState<TaskProgress | null>(null);
    const [connected, setConnected] = useState(false);

    useEffect(() => {
      if (!taskId) return;
      const token = localStorage.getItem('ai_drug_token');
      const wsUrl = `${WS_BASE}/ws/tasks/${taskId}?token=${token}`;
      const ws = new WebSocket(wsUrl);
      // onmessage → setProgress
      // onclose → 指数退避重连（最多 5 次）
      // 3 次失败后降级到 HTTP 轮询（/tasks/{id}/status）
      return () => ws.close();
    }, [taskId]);

    return { progress, connected };
  }
  ```

## 四、Assumptions & Decisions（假设与决策）

### 假设
1. 后端运行环境：conda 环境 `precision-drug-design`（Python 3.11）
2. 前端运行环境：Node.js 18+，Next.js 14+（App Router）
3. 数据库：SQLite（开发） / PostgreSQL（生产），通过 SQLAlchemy 2.0 抽象
4. 所有新增 API 端点遵循现有 `/api/v1` 前缀 + RBAC 鉴权
5. 前端新增页面遵循现有 `frontend/app/workbench/` 目录结构

### 决策
1. **Pinnacle 21 用纯 Python 实现**，不依赖 Java CLI — 便于部署，覆盖 FDA 核心规则
2. **DeepDDI 用规则表 + 靶点重合度**，不集成深度学习模型 — P1 阶段足够，P2 可升级
3. **数据血缘用关系表存储**，不用图数据库 — 与现有 PostgreSQL 兼容，查询深度 ≤ 3 足够
4. **WebSocket 优先 + HTTP 轮询回退** — 后端已实现双模式，前端需对应
5. **知情同意集成到 RBAC**，不单独建权限系统 — 复用现有 `apply_project_visibility` 模式
6. **新增配置项默认启用** — 便于测试，生产环境可通过环境变量关闭
