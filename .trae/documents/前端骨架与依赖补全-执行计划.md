### 已确认的 API 契约（前端对接依据）

| 端点 | 方法 | 入参形式 | 返回结构 |
|---|---|---|---|
| `/api/v1/auth/login` | POST | **query params**: email, password | `{access_token, role, name, email}` |
| `/api/v1/auth/register` | POST | JSON body: `{email, name, password, role, organization?}` | UserResponse |
| `/api/v1/auth/me` | GET | Bearer token | UserResponse |
| `/api/v1/projects` | GET/POST | GET: query(skip,limit)；POST: JSON body | List/ProjectResponse |
| `/api/v1/data` | GET | query: project_id?, data_type? | List[DatasetResponse] |
| `/api/v1/data/upload` | POST | **query params**: project_id, name, data_type, source?；multipart file | DatasetResponse |
| `/api/v1/data/{id}/parse` | POST | path id | StandardResponse |
| `/api/v1/data/{id}/quality` | GET | path id | StandardResponse |
| `/api/v1/targets/discover` | POST | **query params**: project_id, dataset_id?, tier | StandardResponse |
| `/api/v1/targets` | GET | query: project_id?, evidence_grade? | List[TargetResponse] |
| `/api/v1/targets/{id}/repurpose` | POST | path id | StandardResponse |
| `/api/v1/targets/{id}/evidence` | POST | path id | StandardResponse |
| `/api/v1/chat` | POST | JSON body: `{message, project_id, tier}` | ChatResponse |
| `/api/v1/chat/analyze` | POST | **query params**: message, project_id, tier | StandardResponse |
| `/api/v1/chat/tiers` | GET | — | StandardResponse |

**关键注意**：登录、上传、靶点发现、深度分析等使用 query params（非 JSON body），前端 axios 必须用 `params:` 选项。

## Assumptions & Decisions

1. **前端代理策略**：通过 `next.config.js` rewrites 将 `/api/*` 代理到后端 `http://localhost:8000`，开发环境避免 CORS；生产环境用 `NEXT_PUBLIC_API_URL` 环境变量覆盖。
2. **登录态管理**：使用 localStorage 存储 token（简单方案，无 SSR 需求）；Zustand persist 中间件持久化用户信息和当前项目。
3. **Plotly SSR 安全**：用 `dynamic(import, {ssr: false})` 动态加载，避免 Next.js SSR 时 `window is not defined`。
4. **Query Params 严格对接**：后端 login/upload/discover/analyze 等用 query params 而非 JSON body，前端 axios 必须用 `{params: {...}}` 选项传参，已在 `lib/api.ts` 中严格区分。
5. **角色路由守卫**：前端仅做导航可见性过滤（隐藏菜单），真正的权限校验在后端 `require_role` 依赖中。前端守卫失败时显示 403 提示。
6. **不引入 UI 组件库**：仅用 TailwindCSS + 自定义 UI 原语（Card/Button/Badge），保持轻量；如未来需要可平迁到 shadcn/ui。
7. **演示账号**：登录页提示 `sid@ai-drug.com / demo123456`（种子数据预置的 5 角色账号之一）。
8. **Mock 模式提示**：Header 顶部显示黄色横幅"当前为 Mock 模式 — 配置 API key 后切换真实模式"，提醒演示状态。
9. **不实现 Docker 构建前端**：本次仅做前端骨架可本地 `npm run dev` 启动；Docker 化前端留待后续（docker-compose 中已有 frontend service 占位）。
10. **页面数据获取**：使用 `@tanstack/react-query` 的 `useQuery`/`useMutation`，自动处理 loading/error 状态，避免手写 useEffect。
