## 2. 关键决策（D1–D12）

| ID | 决策 | 理由 |
|---|---|---|
| D1 | `/health` 保留无信封；`/api/v1/health` 返回信封；`/api/v1/metrics` 返回 Prometheus 文本（无 system 前缀） | spec 强制路径 |
| D2 | `treatments`/`experiments` 端点保留（规范未列但前端已用） | 业务必需 |
| D3 | `privacy_layer.py` 从 `knowledge/` 迁移到 `privacy/`，旧路径 re-export 保兼容 | spec 期望目录结构 |
| D4 | 前端新页面用 React Query + 现有 `lib/api.ts` 模式 | 与现有架构一致 |
| D5 | `/admin` 拆分为 `admin/{users,llm,audit}` 三个子路由 | 单页过载 |
| D6 | `StandardResponse` 作为 `ApiResponse[Any]` 别名保留 1 版本 | 平滑迁移 |
| D7 | Alembic 在 P1.6 引入（先确保模型可创建表） | 避免阻塞 |
| D8 | EnvelopeMiddleware 仅对 < 1MB JSON 注入 duration_ms | 流式/大响应避免内存压力 |
| D9 | services 层使用 Mock/Real 双模式 | 复用 `settings.USE_MOCK` |
| D10 | 前端路径别名通过 `next.config.js` rewrites 实现 | 双路由并存 |
| D11 | 服务层补全优先实现核心方法，高级特性（PyG/Flower/PySyft）保留降级路径 | 避免阻塞，保证可运行 |
| D12 | LLMRouter/CostTracker/Guardrail 从 orchestrator 拆分，orchestrator 改为调用三者 | spec 期望独立文件 |

## 5. 风险与缓解

| 风险 | 缓解措施 |
|---|---|
| Alembic 与现有 `init_db()` 冲突 | P1.6 末尾引入；保留 `init_db()` 作为 dev fallback |
| 服务层依赖未安装的库（scanpy/cyvcf2/networkx/PyG/Flower/PySyft） | Mock/Real 双模式；Real 模式 try-import 降级 |
| 前端 RDKit.js 体积大（~2MB） | dynamic import + ssr:false；首屏不加载 |
| 双路由别名导致 OpenAPI 文档膨胀 | `include_in_schema=False` 隐藏别名路由 |
| 16 端点改造影响现有前端 | P1 保留 `StandardResponse` 别名；P2 前端适配后再 P4 移除 |
| LLM 成本追踪需要 Redis | 配置项 `LLM_COST_TRACKER_REDIS_URL` 空时降级为内存追踪 |
| 服务层补全工作量大 | 优先核心方法，高级特性保留降级路径（D11） |
| spec 路径偏差（system/metrics → metrics） | P1.3 修正，保留旧路径别名 1 版本 |
