## 阶段 2（P1）：前端 API 客户端 401 SPA 导航修复

### 2.1 修复 `client.ts` 第 49-54 行
- **文件**: [frontend/lib/api/client.ts](file:///g:/软件开发/AI药物/frontend/lib/api/client.ts#L49-L54)
- **问题**: 401 时 `window.location.href = '/'` 触发全页面刷新，丢失 SPA 状态
- **修复**:
  ```typescript
  // 修改前 (第 49-54 行):
  if (status === 401 && typeof window !== 'undefined') {
    localStorage.removeItem('ai_drug_token');
    localStorage.removeItem('ai_drug_user');
    if (window.location.pathname !== '/') {
      window.location.href = '/';
    }
  }

  // 修改后:
  if (status === 401 && typeof window !== 'undefined') {
    localStorage.removeItem('ai_drug_token');
    localStorage.removeItem('ai_drug_user');
    if (window.location.pathname !== '/') {
      // SPA 导航：避免全页面刷新丢失 React Query 缓存
      window.history.replaceState(null, '', '/');
      window.dispatchEvent(new PopStateEvent('popstate'));
    }
  }
  ```
- **说明**: `auth.ts:26` 的 `window.location.href = '/'` 是登出主动跳转，可保留全页面刷新以重置所有状态，仅修复 401 被动跳转

### 3.3 统一修复模式
对每个缺少状态的 useQuery，按以下模式修复：
```typescript
// 修改前:
const { data: items } = useQuery({...});

// 修改后:
const { data: items, isLoading, isError, refetch } = useQuery({...});

// JSX 中添加:
{isLoading && <Loading label="加载中..." />}
{isError && (
  <Card>
    <div className="text-center py-8">
      <AlertCircle className="w-10 h-10 mx-auto mb-2 text-red-400" />
      <p className="text-sm text-gray-500 mb-3">数据加载失败</p>
      <Button size="sm" variant="secondary" onClick={() => refetch()}>重试</Button>
    </div>
  </Card>
)}
{!isLoading && !isError && (!items || items.length === 0) && (
  <Card>
    <div className="text-center py-12 text-gray-400">暂无数据</div>
  </Card>
)}
```
