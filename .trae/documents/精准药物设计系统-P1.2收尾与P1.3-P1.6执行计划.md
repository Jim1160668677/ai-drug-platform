```python
# 改造前
from fastapi import HTTPException
from app.api.v1.schemas import StandardResponse

@router.get("/{id}", response_model=StandardResponse)
async def get_project(id: UUID, db: AsyncSession = Depends(get_db)):
    project = await db.get(Project, id)
    if not project:
        raise HTTPException(status_code=404, detail="项目不存在")
    return StandardResponse(data=project)

# 改造后
from app.core.exceptions import NotFoundError
from app.schemas.common import ApiResponse, success_response

@router.get("/{id}", response_model=ApiResponse[ProjectResponse])
async def get_project(id: UUID, db: AsyncSession = Depends(get_db)):
    project = await db.get(Project, id)
    if not project:
        raise NotFoundError("项目不存在")
    return success_response(ProjectResponse.model_validate(project).model_dump())
```

## 4. 风险与缓解

| 风险 | 缓解措施 |
|---|---|
| `services/__init__.py` 导入失败破坏整个包 | 使用 try/except 包裹每个 import，失败时 log warning 但不中断 |
| 16 端点改造影响现有前端 | 保留 `StandardResponse` 别名（D6）；异常信封格式与现有前端兼容（success: false 已支持） |
| Alembic autogenerate 漏检模型 | 生成后人工审查迁移文件，确保 17 张表 + 5 个新表（quality_reports/docking_results/hypothesis_analyses/target_reports/evidence_items） |
| 双路由别名导致 OpenAPI 文档膨胀 | `include_in_schema=False` 隐藏别名路由 |
| WS 端点与信封中间件冲突 | WS 不经过 EnvelopeMiddleware（中间件仅处理 HTTP） |
| 16 端点改造工作量大 | 优先改造有新增端点的 8 个文件（auth/projects/data/targets/molecules/hypotheses/reports/chat），其余 8 个仅做异常替换 |
