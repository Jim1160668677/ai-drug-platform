## 一、ERR_ABORTED 处理结论（不修复代码）

**现象**：21 条 `net::ERR_ABORTED http://localhost:3000/workbench/xxx?_rsc=xxxxx`

**根因**：Next.js 14 App Router 开发模式按需编译（On-demand Compilation）期间，RSC（React Server Component）请求被瞬时中止。这是开发模式的预期行为，**非真实 bug**：
- 所有路由页面文件均存在
- 生产构建（`next build && next start`）不会出现此现象
- 所有 `Link` 组件已设置 `prefetch={false}`

**验证方法**：
```powershell
# 访问任一报错 URL，返回 HTTP 200 即证明路由正常
curl -I http://localhost:3000/workbench/data
```

**结论**：不做代码修复，在最终交付报告中说明。

### 功能2：多靶点协同分子设计

#### 后端服务

**文件**：`backend/app/services/analyzer/molecule_designer.py`

新增方法 `design_multi_target`：
```python
async def design_multi_target(
    self,
    targets: List[Dict[str, Any]],  # [{target_id, name, binding_site, weight}]
    seed_smiles: Optional[str] = None,
    constraints: Optional[Dict[str, Any]] = None,
    n_molecules: int = 10,
) -> Dict[str, Any]:
    """多靶点协同分子设计
    
    策略：
    1. 为每个靶点单独生成候选分子
    2. 计算每个分子对所有靶点的结合亲和力（Mock: 基于片段匹配）
    3. 按"多靶点综合评分"排序（加权平均亲和力 + 类药性）
    4. 每个分子单独成行，包含 SMILES、理化性质、各靶点亲和力
    """
```

**关键实现**：
- 候选生成：复用 `_generate_by_fragments` / `_generate_by_optimization`
- 亲和力计算（Mock）：基于靶点 binding_site 片段在分子 SMILES 中的匹配度
- 综合评分：`sum(affinity[t] * weight[t] for t in targets) * druglikeness_score`
- 返回结构：`{designed_molecules: [{smiles, properties, target_affinities: {target_id: score}, composite_score}], model_info}`

```python
class HypothesisGenerator:
    """基于前期分析结果自动生成研究假设
    
    数据源：
    1. 差异表达分析结果（DE genes）
    2. 通路富集结果
    3. 分子设计结果（多靶点亲和力）
    4. 治疗方案效果数据
    5. 临床反馈数据
    
    方法：
    - 规则推理：基于阈值生成假设（如 DE gene + 通路富集 → 靶点假设）
    - ML 聚类：对基因/通路聚类，生成组合假设
    """
    
    async def generate(
        self,
        project_id: str,
        context: Optional[Dict[str, Any]] = None,
        max_hypotheses: int = 5,
    ) -> List[Dict[str, Any]]:
        """自动生成假设
        
        Returns:
            [{title, description, supporting_evidence, verification_method, confidence, category}]
        """
```

**关键实现**：
- 规则1：DE gene ∩ 通路富集 → "靶点 X 通过通路 Y 调节疾病 Z"
- 规则2：多靶点高亲和力分子 → "分子 M 可同时抑制靶点 T1/T2"
- 规则3：临床反馈差 + 高不良反应 → "方案 S 需优化剂量"
- 置信度计算：基于证据数量和强度
- 验证方法：自动建议（实验验证/临床回顾/计算模拟）

## 四、通用约定

### 1. 后端约定
- **响应格式**：统一使用 `success_response()` / `ApiResponse[Dict[str, Any]]`
- **鉴权**：所有端点添加 `current_user: User = Depends(get_current_user)`
- **数据库**：`db: AsyncSession = Depends(get_db)`
- **模型注册**：新建模型需在 3 处同步注册（`models/__init__.py`、`db/session.py`、`tests/conftest.py`）
- **外键类型**：使用 `String(36)` 而非 `ForeignKey`（避免 UUID 兼容性问题）
- **日志**：`logger = logging.getLogger(__name__)`
- **Mock 模式**：通过 `settings.USE_MOCK` 控制，Mock 模式下返回预置数据

### 2. 前端约定
- **API 调用**：统一通过 `@/lib/api` 导入
- **数据获取**：使用 React Query `useQuery` / `useMutation`
- **状态管理**：`useAppStore` (Zustand)
- **通知**：`toast.success()` / `toast.error()`
- **UI 组件**：Card, Button, Badge, ProgressBar, PlotlyChart
- **图标**：lucide-react

### 3. 测试约定
- **框架**：pytest + pytest-asyncio
- **覆盖率**：≥ 80%
- **命名**：`test_<功能>_<场景>`
- **Mock**：使用 `settings.USE_MOCK=true` 或直接 Mock 服务
