# 精准药物设计系统 — 代码质量修复计划

> **触发**：用户要求在继续开发前对已完成代码做全面检查并修复所有缺陷。
> **审计范围**：5 个新端点（Agent 1 创建）+ 16 个改造端点（Agent 2 改造）+ 关键支撑文件。
> **审计方法**：Python 导入测试 + 路由诊断 + Agent 深度代码审查。
> **结论**：发现 3 处 P0 阻塞 + 11 处 P1 缺陷 + 12 处 P2/P3 改进。必须先修复 P0/P1 再推进 P1.5+。

### P0-3 文件上传路径遍历

## 2. P1 修复方案（11 处，本阶段）

### P1-1 auth.py 登录密码走 query string

**问题**：`email: str, password: str` 裸 query 参数，密码出现在 URL/日志

```python
from app.api.v1.schemas import LoginRequest  # 已有则用，无则新建

    payload: LoginRequest,  # Body

    user = await authenticate_user(db, payload.email, payload.password)
    ...
```
**注意**：前端 `lib/api.ts` 已用 `params: { email, password }`，需同步改为 Body

### P1-2 projects.py 批量赋值漏洞

**文件**：`backend/app/api/v1/endpoints/projects.py:87-89`
**问题**：`for key, value in payload.items(): setattr(project, key, value)` 允许修改 owner_id/created_at
**修复**：白名单更新：
```python
ALLOWED_UPDATE_FIELDS = {"name", "description", "cancer_type", "stage", "status", "priority"}
update_data = {k: v for k, v in payload.items() if k in ALLOWED_UPDATE_FIELDS}
for key, value in update_data.items():
    setattr(project, key, value)
```

### P1-7 ws.py WebSocket 无认证

**文件**：`backend/app/api/v1/endpoints/ws.py:117-128`
**问题**：任意客户端可订阅任意 task_id，可枚举窃听
**修复**：握手阶段校验 token（query 参数）：
```python
@router.websocket("/ws/tasks/{task_id}")
async def task_progress_ws(websocket: WebSocket, task_id: str):
    # 握手阶段校验 token
    token = websocket.query_params.get("token")
    if not token:
        await websocket.close(code=4001, reason="缺少 token")
        return
    try:
        from app.core.security import decode_access_token
        payload = decode_access_token(token)
        user_id = payload.get("sub")
    except Exception:
        await websocket.close(code=4001, reason="token 无效")
        return
    await websocket.accept()
    ...
```

### P1-8 feedback.py commit 后再抛 NotFoundError

**文件**：`backend/app/api/v1/endpoints/feedback.py:102-115`
**问题**：先 `await db.commit()` 再 `raise NotFoundError`，事务已结束
**修复**：先检查 status 再 commit：
```python
result = await service.submit_feedback(...)
if result.get("status") == "not_found":
    await db.rollback()
    raise NotFoundError("实验不存在")
await db.commit()
return success_response(result)
```

### P1-9 hypotheses.py run_analysis 无状态持久化

**文件**：`backend/app/api/v1/endpoints/hypotheses.py:143-165`
**问题**：创建 `HypothesisAnalysis` 后返回 `"status": "running"`，但模型无 status 字段，无法追踪
**修复**：在 `result_data` 中持久化状态，或添加 `status` 字段到模型：
```python
analysis = HypothesisAnalysis(
    hypothesis_id=hypothesis_id,
    analysis_tier="deep",
    summary="analysis running",
    result_data={"status": "running", "started_at": datetime.utcnow().isoformat()},

db.add(analysis)
await db.commit()
return success_response({
    "analysis_id": str(analysis.id),
    "status": "running",
    "message": "分析已启动，请稍后轮询结果",
})
```

### P1-10 data.py parse_dataset 状态丢失

**文件**：`backend/app/api/v1/endpoints/data.py:97-122`
**问题**：异常抛出后 `get_db` 触发 rollback，`FAILED` 状态被回滚，DB 仍是 `PARSING`
**修复**：在 except 块显式 commit FAILED 状态：
```python
except Exception as e:
    dataset.parse_status = ParseStatus.FAILED
    dataset.parse_error = str(e)[:500]
    await db.commit()  # 显式提交 FAILED 状态
    raise UpstreamError(f"解析失败: {e}", service="parser")
```

