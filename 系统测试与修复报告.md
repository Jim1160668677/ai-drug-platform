## 4. 修复方案与实施记录

### 4.1 P0-1：测试断言修复
- **文件**: `backend/tests/test_services_integration.py`
- **修改**: 将 `status in ("framework_only", "model_load_failed")` 改为 `status in ("rdkit_fallback", "model_load_failed")`
- **原因**: `framework_only` 状态在 `molecule_designer.py` 中不存在，实际返回的是 `rdkit_fallback`
- **验证**: 77/77 测试通过

### 4.2 P0-2：logging.py 作用域 Bug 修复
- **文件**: `backend/app/core/logging.py`
- **修改**: 删除 `except` 块中的 `import sys`，直接使用文件开头已导入的 `sys`
- **原因**: 函数内部的 `import sys` 导致 Python 将 `sys` 视为整个函数的局部变量，使函数开头 `sys.stdout` 引用在 `import sys` 之前触发 `UnboundLocalError`
- **验证**: test_core.py 36/36 通过，test_new_endpoints.py 58/58 通过

### 4.3 P1-1：401 SPA 导航修复
- **文件**: `frontend/lib/api/client.ts`
- **修改**: 将 `window.location.href = '/'` 改为 `window.history.replaceState + dispatchEvent(new PopStateEvent('popstate'))`
- **原因**: 全页面刷新会丢失 React Query 缓存和 SPA 状态
- **验证**: `npx tsc --noEmit` 无新增错误

### 4.4 P1-2：workbench/page.tsx 状态完善
- **文件**: `frontend/app/workbench/page.tsx`
- **修改**:
  - 4 个 useQuery 提取 `isLoading`/`isError`/`refetch` 状态
  - 添加 `statsLoading`/`statsError`/`refetchAll` 聚合状态
  - 统计卡片区域：加载时显示 `LoadingCard` 骨架，错误时显示错误提示+重试按钮
- **验证**: `npx tsc --noEmit` 无新增错误

### 4.5 P1-3：12 个 workbench 子页面错误处理
- **文件**: 12 个 `frontend/app/workbench/*/page.tsx`
- **修改**:
  - 每个 useQuery 提取中添加 `isError` 和 `refetch`
  - 在 `{isLoading ? (` 之前添加 `{isError ? (错误提示) :` 块
  - 使用不依赖额外 import 的内联错误提示组件（`border-red-200 bg-red-50` 样式）
- **涉及文件**:
  1. `consent/page.tsx`
  2. `experiments/page.tsx`
  3. `federated/page.tsx`
  4. `lineage/page.tsx`
  5. `molecules/page.tsx`
  6. `projects/page.tsx`
  7. `privacy/page.tsx`
  8. `targets/page.tsx`
  9. `treatments/page.tsx`（2 个 isLoading 位置）
  10. `hypotheses/page.tsx`（2 个 isLoading 位置）
  11. `efficacy/page.tsx`（2 个查询，summaryError + recordsError）

- **验证**: `npx tsc --noEmit` 无新增错误

## 6. 优化建议与后续工作

### 6.1 短期建议
1. **修复 FASTA 导入测试**：检查 `test_import_fasta` 的断言逻辑，确认是测试数据问题还是导入逻辑问题
2. **修复 LLM 路由器测试**：检查 `TestRouterComplete` 的 Mock 配置，4 个失败可能共享同一根因
3. **前端 vitest 覆盖率**：当前 11 个测试文件覆盖 140 个测试，建议为 workbench 页面添加组件测试

### 6.2 中期建议
1. **统一错误处理组件**：当前 13 个 workbench 子页面使用内联错误提示，建议抽取为可复用的 `<ErrorState />` 组件
2. **React Query 全局错误处理**：在 `QueryClient` 配置中添加全局 `onError` 回调，统一处理网络错误
3. **loading 骨架统一**：当前部分页面使用 `LoadingCard`，部分使用文字"加载中..."，建议统一为骨架屏

### 6.3 长期建议
1. **E2E 测试**：添加 Playwright E2E 测试覆盖关键用户流程（上传数据 → 发现靶点 → 设计分子 → 查看报告）
2. **性能监控**：在前端添加 Web Vitals 监控，在后端添加 APM（如 Sentry）
3. **国际化**：当前所有文案为中文硬编码，建议引入 i18n 框架支持多语言
