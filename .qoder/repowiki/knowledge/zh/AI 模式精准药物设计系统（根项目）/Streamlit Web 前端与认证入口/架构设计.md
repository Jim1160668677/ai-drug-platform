模块由三个文件组成，职责清晰分层：
- `app.py` 是 Streamlit 应用主入口（`streamlit run frontend/app.py`），通过 `st.set_page_config` 配置全局页面，并组合 `render_sidebar`（侧边栏导航）和 `render_home`（首页：未登录显示登录表单 + 系统简介，已登录展示健康状态与快速入口）。子功能页面以 `pages/N_🔖_名称.py` 形式组织，通过 `st.sidebar.page_link` 动态挂载。
- `auth.py` 提供无状态 UI 组件：`render_login_form`（登录/注册双标签页）、`render_user_menu`（右上角登出）、`render_demo_mode_notice`（演示模式提示），所有会话态（`access_token` / `refresh_token` / `user_email` / `api_base_url`）均写入 `st.session_state`。
- `api_client.py` 封装对 FastAPI 后端的 HTTP 调用：`ApiClient` 类复用共享 `httpx.Client`（通过 `@st.cache_resource` 缓存连接池，最大 50 连接、60s 超时），统一注入 Bearer Token、解包 `{success, data, meta}` 信封并在错误时抛出 `RuntimeError`；`cached_get` 借助 `@st.cache_data` + 时间桶 TTL 机制实现请求级缓存；`require_auth` 作为鉴权守卫供子页面复用。
依赖方向：`app.py` → `auth.py` + `api_client.py`；`auth.py` → `api_client.py`；`api_client.py` 仅依赖 httpx 与 streamlit，不反向依赖任何业务页面。
