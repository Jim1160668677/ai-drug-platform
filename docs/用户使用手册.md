## 1. 文档概述

### 1.1 目的

本手册面向「精准药物设计系统」的终端用户，系统性地介绍平台各项功能的使用方式、操作步骤与字段含义，帮助研究人员、医生、数据工程师、首席研究员及创始人快速上手并高效完成精准药物设计全流程工作。

### 1.2 适用对象

| 角色 | 典型场景 |
| --- | --- |
| 研究员（researcher） | 上传组学数据、运行靶点发现、设计分子、提交实验结果 |
| 医生（doctor） | 查看临床数据与靶点报告、参与治疗方案评审 |
| 数据工程师（engineer） | 监控数据质量、查看系统日志、维护数据管道 |
| 首席研究员（chief） | 全量分析结果审阅、模型配置、关键决策建议 |
| 创始人（founder） | 全部权限、模型干预、紧急决策、后台管理 |

### 1.3 阅读建议

- **新用户**：按顺序阅读第 1–4 章完成首次登录与项目创建。
- **研究人员**：重点阅读第 5–9 章（数据 → 靶点 → 分子 → 假设 → 实验）。
- **临床医生**：重点关注第 6、10、12 章（靶点报告、治疗方案、CDISC 导出）。
- **管理员**：重点阅读第 3、13、14 章及 `/admin` 后台说明。

## 2. 系统简介

### 2.1 业务定位

「精准药物设计系统」是一个面向肿瘤精准医疗的端到端药物设计平台，覆盖药物发现全流程：

```
数据接入 → 靶点发现 → 分子设计 → 假设验证 → 干湿闭环 → 治疗方案
   ↑                                                    ↓
   └────────────── 报告导出（CDISC SDTM/ADaM）←──────────┘
```

平台基于患者多组学数据（RNA-seq / scRNA-seq / WES / FASTA），结合 AI 大模型（LLM + RAG）、知识图谱与分子建模，复现 Sid 团队「自然语言提问 → AI 自动分析 → 返回报告 + 代码 + 引用」的研究范式。

### 2.2 核心能力清单

| 能力 | 说明 |
| --- | --- |
| 多组学数据接入 | 支持 RNA-seq / scRNA-seq / WES(VCF) / FASTA / 基因报告 |
| AI 靶点发现 | 突变→注释→通路→证据分级，分级路由（fast_screen / deep_insight） |
| 老药新用 | 扫描 ChEMBL 已获批药物，自动升级证据等级至 I 级 |
| 证据链构建 | 多源证据节点 + 关系边，知识图谱查询 |
| 分子设计 | DeepChem 框架，支持靶点导向与种子 SMILES 约束生成 |
| 类药性评估 | Lipinski 五规则 + Veber 规则 + 类药性评分 |
| 分子对接 | DiffDock 蛋白-配体对接（P2 阶段） |
| 多假设并行 | Hypothesis Sandbox，并行探索多种治疗假设 |
| 干湿闭环 | Dry Prediction → Wet Experiment → 误差反馈 → 模型迭代 |
| 治疗方案优化 | 强化学习多疗法组合优化（P3 阶段） |
| 疗效监测 | 实时疗效、趋势、不良事件追踪 |
| AI 问答 | 自然语言驱动分析，返回报告 + 交互式图表 + Python 代码 |
| 报告导出 | CDISC SDTM（DM/VS/RS/EX/SV）+ ADaM（ADSL/ADRS/ADAE）|

### 2.3 浏览器要求

| 浏览器 | 最低版本 | 备注 |
| --- | --- | --- |
| Google Chrome | 100+ | 推荐使用 |
| Microsoft Edge | 100+ | 推荐 |
| Mozilla Firefox | 100+ | 完全支持 |
| Safari | 16+ | macOS 下支持 |

> 注意：不支持 IE 浏览器。建议屏幕分辨率 ≥ 1366×768，最佳体验为 1920×1080。

## 3. 账户与登录

### 3.1 角色体系

系统采用 5 级 RBAC（基于角色的访问控制）权限模型：

| 角色（英文标识） | 中文名 | 典型职责 | 关键权限 |
| --- | --- | --- | --- |
| `founder` | 创始人 | 全局管控、模型干预、紧急决策、后台管理 | 全部权限 + `model:intervene` + `decision:emergency` + `admin:all` |
| `chief` | 首席研究员 | 审阅全部分析结果、模型配置、决策建议 | `data:read` + `analysis:write` + `model:config` + `audit:read` |
| `researcher` | 研究员 | 在分配项目内上传数据、运行分析、写注释 | `data:read:assigned` + `analysis:run:standard` + `annotation:write` |
| `doctor` | 医生 | 查看临床数据与靶点报告、提供临床建议 | `data:read:clinical` + `target:read` + `clinical:advise` |
| `engineer` | 数据工程师 | 系统日志、数据质量监控、系统配置 | `system:logs` + `quality:read` + `system:config` |

### 3.2 首次登录流程

1. 打开浏览器，访问平台首页 `https://<your-domain>/`，进入登录页。
2. 输入管理员分配的 **邮箱** 与 **初始密码**。
3. 点击「登录」按钮。
4. 登录成功后，系统将 JWT Token 与用户信息存入浏览器 `localStorage`，并自动跳转至 `/dashboard` 全局看板。
5. 首次登录后**强烈建议立即修改密码**（联系管理员或前往 `/admin` 后台）。

<!-- 截图: 登录页表单 -->

**登录字段说明**：

| 字段 | 说明 |
| --- | --- |
| 邮箱 | 注册时填写的邮箱地址，作为账号唯一标识 |
| 密码 | 至少 8 位，建议包含大小写字母、数字、特殊字符 |

### 3.3 密码安全建议

- 密码长度 ≥ 12 位，包含大写字母、小写字母、数字、特殊字符各至少 1 个。
- 不要复用其他网站的密码。
- 每 90 天更换一次密码。
- 不要将密码告知他人或写入代码仓库。
- 离开终端前请点击右上角头像 →「退出登录」。

### 3.4 忘记密码处理

平台当前未提供自助找回密码功能，如忘记密码：

1. 联系系统管理员（founder 角色）或运维团队。
2. 提供您的注册邮箱与身份信息核验。
3. 管理员在 `/admin` 后台为该账户重置初始密码。
4. 收到新密码后，登录并立即修改。

## 4. 工作台总览

### 4.1 进入工作台

登录成功后，默认进入 `/dashboard` 全局看板。点击右上角「进入工作台」按钮，或直接访问 `/workbench`，进入工作台主页。

工作台主页包含四块区域：

1. **页面标题**：显示当前项目名称、癌种、分期。
2. **统计卡片**：展示项目数、数据集数、靶点数、实验数。
3. **快速操作**：上传数据、发现靶点、AI 提问、导出报告四个快捷入口。
4. **当前项目 + 最近活动**：项目详情卡片与最近 5 条实验记录。

<!-- 截图: 工作台主页 -->

### 4.2 项目概念

「项目」（Project）是平台的核心组织单元，对应一名患者或一项研究课题。所有数据、靶点、分子、假设、实验、治疗方案均归属于某个项目。

**项目主要字段**：

| 字段 | 说明 | 示例 |
| --- | --- | --- |
| `name` | 项目名称 | `NSCLC-患者001` |
| `patient_pseudonym` | 患者 pseudonymized 编号 | `PT-2026-0001` |
| `cancer_type` | 癌种 | `非小细胞肺癌 (NSCLC)` |
| `stage` | 分期 | `IIIB` / `IV` |
| `status` | 项目状态 | `active` / `completed` / `paused` |
| `description` | 项目描述 | 自由文本 |

### 4.3 创建新项目

1. 在 `/dashboard` 全局看板页面，进入「项目明细」表格区域。
2. 点击表格上方的「创建项目」按钮（或前往 `/admin` 后台项目管理）。
3. 在弹窗中填写以下必填字段：
   - **项目名称**（必填）：建议格式 `<癌种缩写>-<编号>`，如 `NSCLC-001`。
   - **癌种**（必填）：从下拉列表选择，或自定义输入。
   - **分期**（必填）：如 `I` / `II` / `IIIA` / `IV`。
   - **患者 pseudonym**（可选）：脱敏编号，禁止直接填患者真实姓名。
   - **描述**（可选）：项目背景说明。
4. 点击「创建」，项目即被加入项目列表。

<!-- 截图: 创建项目弹窗 -->

> 注意：根据数据合规要求，**禁止**在项目名或描述中填写患者真实姓名、身份证号、住院号等可识别信息，必须使用 pseudonymized 编号。

### 4.4 项目切换

1. 在工作台任意页面，顶部导航栏会显示当前项目名称。
2. 点击项目名称，弹出项目选择下拉框。
3. 选择目标项目，系统会自动刷新该页面所有数据为对应项目内容。
4. 也可在 `/dashboard` 项目明细表格中点击「进入」按钮，跳转至工作台并切换项目。

## 5. 数据管理（/workbench/data）

数据管理模块负责多组学数据的上传、解析与质量监控。

### 5.1 上传数据文件

**支持的数据类型与文件格式**：

| 数据类型（dataType） | 说明 | 接受格式 |
| --- | --- | --- |
| `rna_seq` | RNA-seq 表达谱 | `.csv` / `.tsv` |
| `scrna_seq` | 单细胞 RNA-seq | `.h5` / `.mtx` |
| `wes` | 全外显子测序（VCF） | `.vcf` |
| `fasta` | 序列文件 | `.fasta` / `.fa` |
| `gene_report` | 基因检测报告 | `.pdf` / `.png` / `.jpg` |

**操作步骤**：

1. 在左侧导航点击「数据管理」，进入 `/workbench/data`。
2. 在「上传新数据」卡片中填写：
   - **数据名称**（必填）：如 `NSCLC RNA-seq`。选中文件后会自动以文件名填充，可修改。
   - **数据类型**（必填）：从下拉框选择对应类型。
   - **数据来源**（可选）：如 `Illumina NovaSeq`、`10x Genomics Chromium`。
   - **文件**（必填）：点击选择本地文件。
3. 点击「上传」按钮，按钮变为加载状态。
4. 上传成功后，下方「数据集列表」自动新增一条记录，初始状态为 `pending`。

<!-- 截图: 数据上传表单 -->

> 文件大小限制：单文件建议 ≤ 500 MB。大文件请先联系数据工程师协助通过对象存储上传。

### 5.2 解析状态查看

数据集的解析状态（`parse_status`）流转如下：

| 状态 | Badge 显示 | 含义 |
| --- | --- | --- |
| `pending` | planned（灰） | 已上传，等待解析 |
| `parsing` | running（蓝） | 正在解析 |
| `completed` | completed（绿） | 解析完成 |
| `failed` | failed（红） | 解析失败 |

**触发解析**：

1. 在数据集列表中找到状态为 `pending` 的记录。
2. 点击该行「解析」按钮。
3. 系统调用对应 parser，状态变为 `parsing`。
4. 解析完成后状态变为 `completed`，可点击「质量」按钮查看质量报告。
5. 若解析失败，状态变为 `failed`，请查看错误详情并联系数据工程师。

<!-- 截图: 数据集列表与状态 Badge -->

### 5.3 质量指标解读

点击数据集行的「质量」按钮，打开「数据质量报告」弹窗。

**质量指标（quality_metrics）常见字段**：

| 指标 | 含义 | 参考范围 |
| --- | --- | --- |
| `total_reads` | 总读段数 | ≥ 20M（RNA-seq） |
| `mapping_rate` | 比对率 | ≥ 80% |
| `q30_rate` | Q30 比例 | ≥ 85% |
| `gc_content` | GC 含量 | 40%–60% |
| `duplicate_rate` | 重复率 | ≤ 20% |
| `n_genes` | 检测到基因数 | ≥ 15000（RNA-seq） |

弹窗同时显示：
- **指标数值卡片**：每个指标一张卡片，数值保留 3 位小数。
- **解析摘要**（JSON 格式）：原始 parser 输出。
- **质量指标柱状图**：Plotly 渲染的可视化图表。

<!-- 截图: 质量报告弹窗 -->

### 5.4 数据查看与筛选

1. 数据集列表按创建时间倒序展示。
2. 表格列：名称、类型、格式、大小、状态、操作。
3. 当前支持按项目维度自动筛选（顶部导航的项目切换器）。
4. 后端支持 `project_id` 与 `data_type` 查询参数筛选，前端按项目筛选已默认开启。

## 6. 靶点发现（/workbench/targets）

### 6.1 AI 靶点发现流程

平台提供两级分析能力（tier）：

| 层级 | 名称 | 成本 | 时长 | 技术栈 | 适用场景 |
| --- | --- | --- | --- | --- | --- |
| `fast_screen` | 快速筛查 | < $5 | < 5 min | 统计分析 + 规则引擎 + 小模型 | 靶点初筛、批量数据扫描 |
| `deep_insight` | 深度洞察 | < $20 | < 30 min | LLM + RAG + 网络分析 + 分子建模 | 候选靶点深度分析、分子设计 |

**操作步骤**：

1. 进入 `/workbench/targets` 靶点发现页面。
2. 点击右上角「发现靶点」按钮，弹出分析层级选择对话框。
3. 选择分析层级：
   - 快速筛查：用于初步扫描，成本低、速度快。
   - 深度洞察：用于已筛选候选靶点的深度分析。
4. 点击「开始发现」按钮。
5. 后端执行流程：突变检测 → 功能注释 → 通路富集 → 证据分级。
6. 完成后页面自动刷新靶点卡片列表。

<!-- 截图: 发现靶点层级选择弹窗 -->

### 6.2 证据分级

每个靶点根据证据强度分为 4 级：

| 等级 | Badge 颜色 | 含义 |
| --- | --- | --- |
| I 级 | 绿 | 已获批药物靶点（FDA/NMPA） |
| II 级 | 黄 | 处于临床试验阶段 |
| III 级 | 橙 | 临床前研究证据 |
| IV 级 | 红 | 推测性证据 |

**筛选操作**：页面顶部「证据等级筛选」下拉框，支持按 I / II / III / IV 级筛选靶点。

### 6.3 置信度评分解读

每个靶点卡片显示置信度评分（`confidence_score`，0–1）：

- 评分 ≥ 0.8：高置信度，优先推荐。
- 评分 0.5–0.8：中等置信度，建议进一步验证。
- 评分 < 0.5：低置信度，仅供参考。

评分以百分比形式显示（如 `85.0%`），并配进度条可视化。

### 6.4 老药新用（drug repurposing）

1. 在靶点卡片上点击「老药新用」按钮。
2. 系统调用 `DrugRepurposer`，扫描 ChEMBL 数据库中已获批药物。
3. 若找到候选药物：
   - 靶点的 `approved_drugs` 字段填入候选列表。
   - 靶点证据等级自动升级为 **I 级**。
4. 弹窗或卡片底部显示候选药物数量。

<!-- 截图: 老药新用结果 -->

### 6.5 证据链查看

1. 在靶点卡片上点击「证据链」按钮。
2. 系统调用 `EvidenceChainBuilder` 构建多源证据网络。
3. 弹出「证据链」弹窗，展示：
   - **统计卡片**：总证据数、I 级 / II 级 / III+IV 级分布。
   - **证据摘要**：自然语言概述。
   - **证据节点列表**：每条证据的类型、标签、临床意义、适应症、阶段、状态。
   - **关系边列表**：节点间关系（如 `target —inhibits→ drug`），最多展示 20 条。

<!-- 截图: 证据链弹窗 -->

### 6.6 知识图谱查询

点击靶点卡片「详情」按钮，打开靶点详情抽屉，以 JSON 形式查看靶点完整信息，包括：

- `gene_symbol`：基因符号（如 `EGFR`）
- `gene_name`：基因全名
- `evidence_grade`：证据等级
- `confidence_score`：置信度
- `approved_drugs`：已获批药物列表
- `variant_info`：变异信息列表
- `evidence_chain`：完整证据链（节点 + 边）

JSON 内容支持复制粘贴，便于二次分析。

## 7. 分子库（/workbench/molecules）

### 7.1 分子设计

**操作步骤**：

1. 进入 `/workbench/molecules` 分子库页面。
2. 点击右上角「设计分子」按钮，打开「分子设计（P2 DeepChem 框架）」弹窗。
3. 填写参数：
   - **靶点 ID**（可选）：从靶点列表复制 UUID，如 `96d78aa1-b4f2-4bbd-8d63-43431bca1941`。
   - **种子 SMILES**（可选）：如 `CCO` 或 `COC1=CC=C(C=C1)N`，作为分子生成起点。
   - **分子量上限**（默认 500）：符合 Lipinski 五规则建议。
   - **LogP 上限**（默认 5）：符合 Lipinski 五规则建议。
4. 点击「开始设计」按钮。
5. 设计完成后，弹窗下方显示结果：
   - **成功**：列出 `designed_molecules` 数组，每个分子含 SMILES 与预测毒性。
   - **框架未就绪**：显示 `model_info.status` 与 `required_packages`，需后端安装 DeepChem 相关依赖。

<!-- 截图: 分子设计弹窗 -->

### 7.2 类药性评估

**操作步骤**：

1. 点击右上角「评估 SMILES」按钮。
2. 在「类药性评估（Lipinski 五规则）」弹窗中输入 SMILES 字符串。
   - 示例：`CCO`（乙醇）、`CC(=O)Oc1ccccc1C(=O)O`（阿司匹林）、`CN1C=NC2=C1C(=O)N(C(=O)N2C)C`（咖啡因）。
3. 点击「评估」按钮。
4. 结果区显示：

**评估指标卡片**（共 8 个）：

| 指标 | 字段 | Lipinski / Veber 阈值 |
| --- | --- | --- |
| 分子量 (MW) | `mw` | ≤ 500 |
| LogP | `logp` | ≤ 5 |
| 氢键供体 (HBD) | `hbd` | ≤ 5 |
| 氢键受体 (HBA) | `hba` | ≤ 10 |
| 可旋转键 | `rotatable_bonds` | ≤ 10（Veber） |
| 极性表面积 (TPSA) | `tpsa` | ≤ 140（Veber） |
| 环数 | `n_rings` | — |
| 芳香环数 | `n_aromatic_rings` | — |

> 超出阈值的指标卡片会以红底高亮显示。

**规则通过情况**：

- **Lipinski 五规则**：通过 / 违反（绿底 / 红底）
- **Veber 规则**：通过 / 违反
- **类药性评分**：0–100，分数越高类药性越好

**违反规则明细**：列出具体违反项，如 `MW > 500`、`LogP > 5`。

**分子属性雷达图**：双层雷达图，蓝色为实际值，红色虚线为 Lipinski 上限。

<!-- 截图: 类药性评估结果 -->

### 7.3 分子对接结果查看

1. 在分子列表中点击某分子的「详情」按钮。
2. 详情弹窗显示：
   - **SMILES**：完整 SMILES 字符串。
   - **属性卡片**：分子量、LogP、InChI Key、ChEMBL ID。
   - **Lipinski 五规则雷达图**：MW / LogP / HBD / HBA / PSA 五维雷达。
   - **对接结果**（JSON 格式）：包含对接得分、构象等信息。

> 分子对接（DiffDock）为 P2 阶段功能，需后端部署 DiffDock 服务。前端可通过 `/molecules/{id}/dock` 接口触发对接。

### 7.4 分子属性表

分子列表表格展示以下列：

| 列名 | 字段 | 说明 |
| --- | --- | --- |
| 名称 | `name` | 分子名称，未命名为「未命名」 |
| SMILES | `smiles` | 等宽字体显示，过长自动截断 |
| 分子量 | `molecular_weight` | 保留 1 位小数 |
| LogP | `logp` | 保留 2 位小数 |
| 来源 | `is_approved` / `source` | 已获批（绿）/ 设计中（灰）+ 来源标记 |

## 8. 多假设并行（/workbench/hypotheses）

「多假设并行」（Hypothesis Sandbox）允许创始人与首席研究员为同一项目并行探索多种治疗假设，对比择优。

### 8.1 创建假设

**操作步骤**：

1. 进入 `/workbench/hypotheses` 多假设并行页面。
2. 点击右上角「新建假设」按钮。
3. 在弹窗中填写：
   - **假设名称**（必填）：如 `H1 - EGFR 通路抑制`。
   - **描述**：假设的核心论断，如 `针对 EGFR L858R 突变，使用第三代 TKI 单药抑制`。
   - **机制**：如 `酪氨酸激酶抑制`。
   - **策略**：如 `第三代 TKI 单药`。
4. 点击「创建」按钮。

<!-- 截图: 新建假设弹窗 -->

### 8.2 假设分析与对比

**单假设分析**：

1. 在假设卡片上点击「分析」按钮。
2. 系统调用靶点发现引擎，对该假设执行分析。
3. 假设状态变为 `analyzing`，分析完成后变为 `completed`。
4. 卡片底部显示分析结果，包括置信度（`analysis_result.confidence`）与靶点列表（`target_list`）。

**多假设对比**：

1. 在多个假设卡片的复选框中勾选至少 2 个假设。
2. 点击右上角「对比 (N)」按钮（N 为已选数量）。
3. 弹出「假设对比」弹窗，以 JSON 形式并排展示：
   - 假设 ID、名称、状态
   - 靶点列表（`targets`）
   - 分析结果摘要（`result_summary`）
   - 是否强制深度分析（`forced`）

<!-- 截图: 假设对比弹窗 -->

### 8.3 假设状态流转

| 状态 | Badge | 含义 |
| --- | --- | --- |
| `planned` | planned（灰） | 已创建，待分析 |
| `analyzing` | running（蓝） | 分析中 |
| `completed` | completed（绿） | 分析完成 |
| `merged` | completed（绿） | 已合并入其他假设 |

**创始人强制深度分析**：创始人角色可在分析时通过 `force_deep=true` 强制对假设使用 `deep_insight` 层级分析，需填写 `force_reason`（强制理由）。

**假设合并**：通过 `/hypotheses/{id}/merge?target_hypothesis_id=...` 接口将一个假设合并入另一个假设，原假设状态变为 `merged`。

## 9. 干湿闭环（/workbench/experiments）

干湿闭环（Dry-Wet Loop）是平台核心创新：Dry Prediction → Wet Experiment → Error Analysis → Model Update → 下一轮迭代。

### 9.1 提交实验

**支持的实验类型（exp_type）**：

| 类型 | 说明 |
| --- | --- |
| `binding` | 分子结合实验（SPR / ITC） |
| `functional` | 功能性实验（细胞活力、酶活） |
| `admet` | ADMET（吸收/分布/代谢/排泄/毒性） |
| `in_vivo` | 体内实验（动物模型） |
| `in_vitro` | 体外实验（细胞系） |

**创建实验**：通过 `/experiments` POST 接口创建，需指定 `project_id`、`name`、`exp_type`，可选 `target_id`、`molecule_id`、`treatment_id`、`config`、`lab_source`。

### 9.2 实验结果录入

**操作步骤**：

1. 进入 `/workbench/experiments` 干湿闭环页面。
2. 在「迭代时间线」卡片中，按迭代轮次分组查看实验列表。
3. 找到状态为 `planned` 或 `running` 的实验，点击「提交结果」按钮。
4. 在弹窗中填写：
   - **实测值**（必填）：数值类型，保留 3 位小数，如 `0.85`。
   - **备注**：实验观察、异常情况等文字说明。
5. 点击「提交并触发反馈」按钮。
6. 系统执行：
   - 将结果写入 `experiment.result`。
   - 设置 `experiment.status = completed`、`experiment.success`。
   - 调用 `FeedbackLoop.apply_feedback(exp)`，触发模型权重更新。
7. 页面下方「反馈分析结果」卡片以 JSON 形式展示反馈详情。

<!-- 截图: 提交实验结果弹窗 -->

### 9.3 反馈到模型（更新权重）

提交实验结果后，`FeedbackLoop` 会：

1. 计算预测值（Dry Prediction）与实测值（Wet Experiment）的误差。
2. 根据误差调整相关模型权重。
3. 标记 `experiment.feedback_applied = True`。
4. 返回反馈详情（包含误差、权重调整、建议等）。

反馈后的实验卡片会显示「已反馈」Badge，表明该实验已驱动模型迭代。

### 9.4 迭代轮次管理

实验按 `iteration` 字段分组展示：

- 每个迭代轮次以圆形序号标识（1、2、3...）。
- 该轮次下所有实验以时间线形式列出。
- 每条实验显示：名称、类型、实验室来源、状态、关键结果指标（前 4 个字段）。
- 通过 `/experiments/loop-status?project_id=...` 可查看闭环状态汇总：
  - `total_experiments`：实验总数
  - `completed`：已完成数
  - `successful`：成功数
  - `feedback_applied`：已反馈数
  - `max_iteration`：当前最大迭代轮次

<!-- 截图: 迭代时间线 -->

> 数据工程师可通过 `/experiments/lims-import` 接口从 LIMS 系统批量导入实验数据。

## 11. AI 问答（/workbench/chat）

AI 问答模块复现 Sid 团队研究范式：研究者提问 → AI 自动执行分析 → 返回报告 + 代码 + 引用。

### 11.1 提问方式

1. 进入 `/workbench/chat` AI 问答页面。
2. 在顶部选择分析层级（`TierSelector`）。
3. 在输入框中输入自然语言问题，例如：
   - `EGFR L858R 突变在 NSCLC 中的临床意义是什么？`
   - `帮我分析这个患者的潜在耐药机制`
   - `推荐针对 BRCA1 突变的老药新用候选`
4. 点击「发送」按钮（或按 Enter 键）。
5. AI 响应会以消息列表形式追加显示。
6. 若已选择项目，可点击「深度分析」按钮触发 `analyze_with_nl` 接口。

<!-- 截图: AI 问答界面 -->

### 11.2 快速筛查 vs 深度洞察

通过 `TierSelector` 切换分析层级：

| 层级 | 名称 | 技术栈 | 适用场景 |
| --- | --- | --- | --- |
| `fast_screen` | 快速筛查 | 统计分析 + 规则引擎 + 小模型 | 靶点初筛、批量数据扫描 |
| `deep_insight` | 深度洞察 | LLM + RAG + 网络分析 + 分子建模 | 候选靶点深度分析、分子设计 |

层级说明可通过 `/chat/tiers` 接口获取，包含 `max_cost_usd` 与 `max_duration_sec` 上限。

### 11.3 RAG 知识检索

深度洞察模式下，AI 会通过 RAG（Retrieval-Augmented Generation）检索知识库：

1. 从内置文献库、药物数据库、临床指南中检索相关条目。
2. 将检索结果作为上下文喂给 LLM。
3. 在最终回答中标注引用来源。
4. 支持后续追问，保持上下文连续性。

### 11.4 Markdown 渲染与代码高亮

- AI 回答支持完整 Markdown 渲染：标题、列表、表格、引用块、行内代码、代码块。
- 代码块支持 Python / SQL / JSON 等语法高亮。
- 分析结果卡片（`AnalysisResultCard`）展示：
  - **结论**：自然语言总结
  - **交互式图表**：Plotly 渲染的可视化
  - **AI 生成的分析代码**：可直接复制运行的 Python 代码
  - **引用文献**：支持点击跳转

### 12.2 CDISC SDTM 标准导出

**SDTM（Study Data Tabulation Model）** 是 FDA/NMPA 提交临床试验数据的标准格式。

**操作步骤**：

1. 在「SDTM 导出」卡片中点击「生成」按钮。

| 域 | 含义 | 关键字段 |

### 12.4 报告字段说明

- 字段值若为空，表格中以 `—` 显示。
- 表格仅展示前 5 条，完整数据请下载 CSV。

## 14. 通知与错误提示

### 14.1 Toast 通知

平台使用右上角 Toast 通知反馈操作结果，共 4 种类型：

| 类型 | 图标 | 颜色 | 触发场景 |
| --- | --- | --- | --- |
| `success` | CheckCircle | 绿底 | 操作成功（如上传成功、提交成功） |
| `error` | XCircle | 红底 | 操作失败（如登录失败、接口报错） |
| `warning` | AlertTriangle | 黄底 | 警告（如即将超限、数据不全） |
| `info` | Info | 蓝底 | 一般提示（如后台任务进行中） |

**通知结构**：
- 主消息（`message`）：简短文字描述。
- 详情（`detail`，可选）：详细说明，自动换行。
- 关闭按钮：点击右上角 X 手动关闭。

通知从屏幕右上角滑入，自动消失时间由系统配置。鼠标悬停可查看完整内容。

<!-- 截图: Toast 通知示例 -->

### 14.2 表单校验错误

- **必填字段**：未填写时提交按钮置灰（`disabled`），无法点击。
- **格式校验**：如 SMILES 格式错误，提交后红色文字提示 `上传失败：<详情>`。
- **数值范围**：如分子量上限输入负数，前端会拦截。
- **后端校验**：返回 422 时，详情中列出具体字段错误。

### 14.3 错误边界（渲染异常处理）

- 若前端组件渲染异常，页面会显示「加载失败」卡片，提示 `无法获取全局看板数据，请确认后端服务正常`。
- 网络错误（如后端服务未启动）会显示 `AlertCircle` 红色图标与提示。
- 遇到错误边界时，请：
  1. 刷新页面重试。
  2. 检查网络连接。
  3. 联系数据工程师查看后端服务状态。

## 15. 常见问题 FAQ

### Q1：上传失败怎么办？

**A**：
1. 查看红色错误提示 `上传失败：<详情>`，确认错误原因。
2. 检查文件格式是否在支持列表（CSV/TSV/H5/MTX/VCF/FASTA/FA/PDF/PNG/JPG）。
3. 检查文件大小是否超过 500 MB 限制。
4. 检查当前是否已选择项目（数据必须归属某项目）。
5. 检查「数据名称」是否填写。
6. 联系数据工程师查看后端 `/data/uploads/` 目录权限。

### Q2：解析卡在 pending 怎么办？

**A**：
1. `pending` 表示已上传但未触发解析，请点击该数据集行的「解析」按钮。
2. 若点击「解析」后状态变为 `parsing` 长时间不变化：
   - 大文件解析耗时较长（scRNA-seq 可能需 10+ 分钟）。
   - 联系数据工程师查看后端 parser 任务队列。
3. 若状态变为 `failed`，查看错误详情并联系支持。

### Q3：AI 问答无响应？

**A**：
1. 检查网络连接是否正常。
2. 切换分析层级为 `fast_screen`（成本低、响应快）。
3. 后端 LLM 服务可能限流，稍后重试。
4. 若长时间无响应，联系数据工程师查看 LLM 服务状态。
5. 检查当前账户是否有 `analysis:run:standard` 权限。

### Q4：实验结果无法提交？

**A**：
1. 确认实验状态为 `planned` 或 `running`（`completed` 状态无法再提交）。
2. 确认「实测值」字段已填写且为数字。
3. 检查网络连接。
4. 后端 `FeedbackLoop` 可能因模型服务异常导致反馈失败，联系数据工程师排查。

### Q5：角色权限不足怎么办？

**A**：
1. 查看错误详情 `权限不足，需要角色: [...]`。
2. 联系创始人（founder）或管理员在 `/admin` 后台调整您的角色。
3. 研究员仅能访问分配项目内的数据；如需跨项目，请申请首席研究员角色。
4. 强制深度分析等高风险操作仅创始人可执行。

### Q6：数据导出格式有哪些？

**A**：
- **SDTM CSV**：通过 `/reports` 页面「下载 CSV」按钮，文件名 `sdtm_<project_id>.csv`。
- **ADaM JSON**：通过 `/reports` 页面生成，包含 ADSL / ADRS / ADAE 数据集。
- **原始 JSON**：靶点详情、分子详情、对比结果等均可通过弹窗查看 JSON。
- **图表**：Plotly 图表支持右侧工具栏下载 PNG / SVG。

### Q7：浏览器兼容性？

**A**：
- 支持 Chrome 100+ / Edge 100+ / Firefox 100+ / Safari 16+。
- 不支持 IE。
- 若图表（Plotly）显示异常，请升级浏览器至最新版。
- 启用浏览器 JavaScript 与 Cookie。
- 不要使用无痕模式（会影响 `localStorage` 持久化）。

### Q8：会话超时怎么办？

**A**：
1. JWT Token 默认有效期由后端 `JWT_ACCESS_TOKEN_EXPIRE_MINUTES` 配置（通常 60 分钟）。
2. Token 过期后，接口返回 401，页面自动跳转回登录页。
3. 重新登录即可恢复。
4. 频繁超时可联系管理员延长有效期，但出于安全考虑不建议过长。

### Q9：项目无法删除？

**A**：
1. 当前前端未提供项目删除入口，避免误操作。
2. 如确需删除，联系创始人通过 `/admin` 后台或后端接口执行。
3. 推荐做法：将项目状态改为 `paused` 或 `archived`，保留数据可追溯性。
4. 删除项目会级联删除其下所有数据集、靶点、分子、假设、实验、治疗方案。

### Q10：分子 SMILES 格式错误？

**A**：
1. 评估 SMILES 时若返回 `error`，说明格式不合法。
2. SMILES 必须符合规范，常见示例：
   - 乙醇：`CCO`
   - 阿司匹林：`CC(=O)Oc1ccccc1C(=O)O`
   - 咖啡因：`CN1C=NC2=C1C(=O)N(C(=O)N2C)C`
3. 检查括号是否配对、原子符号是否大写、键符号是否合法。
4. 推荐使用 RDKit 或 PubSketch 在线工具验证 SMILES。

### Q11：靶点置信度评分如何计算？

**A**：置信度评分（0–1）综合考虑：
- 变异频率与功能影响预测
- 证据等级（I 级贡献最高，IV 级最低）
- 通路富集显著性
- 文献支持度（RAG 检索）
- 临床试验阶段

具体算法详见后端 `TargetIdentifier` 服务。

### Q12：如何查看系统日志？

**A**：数据工程师角色（`engineer`）拥有 `system:logs` 权限，可在 `/admin` 后台查看：
- API 访问日志
- 解析任务日志
- LLM 调用日志
- 模型反馈日志

## 16. 快捷键与操作技巧

### 16.1 通用快捷键

| 快捷键 | 功能 |
| --- | --- |
| `Ctrl/Cmd + Enter` | 在 AI 问答输入框中发送消息 |
| `Esc` | 关闭当前打开的弹窗 |
| `Ctrl/Cmd + S` | （在表单页）保存当前草稿（部分页面支持） |
| `F5` / `Ctrl/Cmd + R` | 刷新当前页面（数据重新拉取） |
| `Ctrl/Cmd + L` | 快速定位到登录页（未登录时） |

### 16.2 操作技巧

1. **项目切换**：在任意工作台页面顶部点击项目名即可切换，无需返回仪表盘。
2. **批量操作**：多假设对比页面支持勾选多个假设一次性对比。
3. **JSON 复制**：靶点详情、分子对接结果等 JSON 弹窗支持选中复制。
4. **图表交互**：Plotly 图表支持鼠标悬停查看数值、双击复位、滚轮缩放、工具栏下载。
5. **状态筛选**：靶点页面支持按证据等级筛选；实验页面按迭代轮次自动分组。
6. **CSV 下载**：报告页面的 SDTM CSV 文件可直接用 Excel 打开。
7. **Tier 选择**：AI 问答页面顶部可快速切换 fast_screen / deep_insight，影响成本与响应时长。
8. **错误重试**：上传/解析失败时，可直接点击对应按钮重试，无需刷新页面。
